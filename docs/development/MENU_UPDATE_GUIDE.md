# 菜单配置更新说明

## 📋 更新内容

### 新增菜单项

在"用户权限管理"子菜单下新增了3个菜单项：

1. **多因素认证** (`/mfa`)
   - 图标: Lock（锁图标）
   - 页面: 多因素认证设置页面
   - 功能: MFA启用/禁用、备用码管理

2. **会话管理** (`/session`)
   - 图标: Monitor（显示器图标）
   - 页面: 会话管理页面
   - 功能: 查看和管理用户会话

3. **审计日志** (`/audit-log`)
   - 图标: DocumentChecked（文档勾选图标）
   - 页面: 审计日志查询页面
   - 功能: 日志查询、统计、导出

### 菜单结构

```
用户权限管理
├── 用户管理
├── 角色管理
├── 权限管理
├── 🔒 多因素认证 (新增)
├── 🖥️ 会话管理 (新增)
└── 📋 审计日志 (新增)
```

## 🚀 使用方法

### 访问新功能

1. **多因素认证**
   - 菜单路径: 用户权限管理 → 多因素认证
   - URL路径: `/mfa`
   - 功能: 设置和管理MFA认证

2. **会话管理**
   - 菜单路径: 用户权限管理 → 会话管理
   - URL路径: `/session`
   - 功能: 查看和管理用户会话

3. **审计日志**
   - 菜单路径: 用户权限管理 → 审计日志
   - URL路径: `/audit-log`
   - 功能: 查询和统计审计日志

### 权限控制
- 所有新菜单项都需要用户登录才能访问
- 通过路由守卫进行权限验证
- 未登录用户会被重定向到登录页

## 📝 注意事项

### 1. 图标依赖
新添加的图标来自Element Plus图标库，确保已正确安装：
```bash
npm install @element-plus/icons-vue
```

### 2. 路由配置
确保路由配置正确，避免路径冲突：
- 新路由路径不与现有路由冲突
- 路由名称唯一
- 组件路径正确

### 3. 权限控制
新菜单项都设置了`requiresAuth: true`，确保：
- 用户必须登录才能访问
- 未登录用户会被正确重定向
- 登录后可以正常访问
